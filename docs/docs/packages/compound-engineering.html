Compound Engineering

Official Compound Engineering from Every, installed through the upstream Pi target converter.

What it does

LazyPi installs the official @every-env/compound-plugin package from Every and runs its Pi target converter. That upstream tool generates Pi-compatible skills, extensions, and agent guidance directly into your Pi config directory.

In practice, this gives Pi the official Compound Engineering reviewer and workflow skill set — including entry points like /skill:ce-plan, /skill:ce-work, /skill:ce-review, /skill:ce-brainstorm, and /skill:ce-compound — plus the compatibility layer required for subagents and MCP-style access.

LazyPi also normalizes the upstream colon-style Compound skill IDs into Pi-compatible hyphenated IDs (for example ce:plan → ce-plan) so current Pi releases do not emit skill-name warnings. That normalization is re-applied on every Compound reinstall or lazypi update.

After installing LazyPi, restart or /reload Pi so the generated skills and extensions are loaded into the current session.

Bun requirement

The official Every installer is a Bun-based CLI. LazyPi checks for bun before attempting this package. If Bun is missing, LazyPi will warn you and continue installing everything else, but Compound Engineering will be skipped.

Why it's included

Compound Engineering remains one of the strongest AI-assisted development methodologies available, and using the upstream package keeps LazyPi aligned with Every's official implementation instead of a third-party fork or repackaging.

Removal

LazyPi tracks the files it generates for the official Pi target, so npx @robzolkos/lazypi remove compound removes the generated Compound Engineering files cleanly instead of leaving converted skills behind.

What shows up in Pi

The official Pi target installs a large skill library and supporting tooling rather than a single npm-style Pi package manifest entry. In practice, you get the core Compound Engineering workflows plus a broad bench of reviewer, researcher, planning, testing, and shipping skills that those workflows can call.
